# v0.3.0-beta.1 — Digger Audit Triage Beta

## Summary

Digger Audit Triage Beta is the first public beta release of Digger's local-first, evidence-gated audit triage workflow for EVM and Solana security researchers.

Digger is deterministic at the evidence layer and agentic/LLM-assisted by design. The beta runs with provider `none` — no model calls required. Models may assist with hypotheses, ranking, explanation, and report drafting, but model output never becomes a finding.

**AI can suspect. Digger proves.**

## Highlights

- `digger audit-triage` CLI workflow
- Repo-derived AuditTriagePacket JSON output
- EVM function-level source triage (signatures, visibility, auth, mutation, call signals)
- Solana instruction/account-level source triage (instructions, Signer, mut, CPI, seeds, has_one)
- Solana account struct detection with field-level analysis
- Evidence-specific missing evidence with source path references
- Conservative candidate hypotheses tied to specific functions/instructions
- Conservative proof tasks tied to specific functions/instructions
- Fuzz maturity scanning (Foundry/Echidna/Medusa infrastructure)
- Foundry fuzz artifact parsing
- Human summary output
- Public beta README, walkthrough, and feedback templates
- LLM-assisted architecture boundary documented

## Agentic / LLM-assisted architecture

Digger's architecture has two layers:

1. **Deterministic evidence layer** — parses source code, constructs graphs, runs detectors, produces evidence packets. Works with provider `none`.

2. **Agentic / LLM-assisted layer** — models may propose hypotheses, rank attack surfaces, explain evidence, suggest invariants, and draft report text. Model outputs are untrusted until grounded through the evidence layer.

Models may: summarize, propose hypotheses, rank surfaces, explain evidence, suggest invariants, suggest fuzz/proof tasks, draft reports.

Models must NOT: decide findings, create EvidenceRun objects, bypass evidence gates, override validation failures, make severity decisions as truth.

`decide_valid_finding` is forbidden by architecture.

## Safety boundaries

- No model-to-finding shortcut
- No final findings by default
- No fuzzer execution by default
- No exploit execution
- No source mutation
- No hidden network calls from `audit-triage`
- `is_finding: false` in all packets
- Model output cannot create findings

## Known limitations

- Heuristic text scanning only — no AST or dataflow analysis
- No compilation or type checking of target code
- EVM modifier detection may miss complex multi-line patterns
- Solana CPI detection is pattern-based, not type-aware
- No cross-function dataflow analysis
- LLM-assisted layer is schema/policy baseline, not live provider runtime in this beta
- Audit triage is not a complete audit

## Quickstart

```bash
cargo build --release -p digger-cli

# EVM
./target/release/digger audit-triage --path examples/evm-basic --chain evm --json --output triage.json

# Solana
./target/release/digger audit-triage --path examples/solana-basic --chain solana --json --output triage.json

# Human summary
./target/release/digger audit-triage --path examples/evm-basic --chain evm
```
