# v0.3.0-beta.2 — Digger Audit Triage Beta

## Summary

`v0.3.0-beta.2` is the architecture-corrected public beta release point for Digger Audit Triage Beta.

It supersedes `v0.3.0-beta.1` for public beta launch because it includes the corrected agentic/LLM-assisted, evidence-gated architecture documentation from PR #71.

**AI can suspect. Digger proves.**

## What changed since v0.3.0-beta.1

- README now correctly frames Digger as evidence-gated AI-assisted blockchain security infrastructure
- Added `docs/product/LLM-ASSISTED-BETA-BOUNDARY.md` — implemented vs schema/policy baseline
- Added `docs/product/PUBLIC-BETA-REPO-EXTRACTION.md` — safe public beta extraction plan
- Updated public beta launch docs and launch post with correct architecture framing
- Updated release notes and GitHub release body with agentic/LLM-assisted architecture
- Clarified what is implemented vs what is schema/policy baseline
- Confirmed no unsafe model runtime was added

## What is implemented

- `digger audit-triage` CLI — deterministic packet generation
- EVM function-level source triage (signatures, visibility, auth, mutation, call signals)
- Solana instruction/account-level source triage (instructions, Signer, mut, CPI, seeds, has_one)
- Solana account struct detection with field-level analysis
- Repo intelligence scanning
- Fuzz maturity scanning (Foundry/Echidna/Medusa infrastructure)
- Foundry fuzz artifact parsing
- Evidence-specific missing evidence with source path references
- Conservative candidate hypotheses tied to specific functions/instructions
- Conservative proof tasks tied to specific functions/instructions
- Human summary output
- Public beta docs, walkthrough, and feedback templates

## Schema/policy baseline (defined, not live runtime)

- Model Provider Abstraction (`digger.model_provider_config.v1`)
- LLM Firewall (`digger.llm_firewall_policy.v1`)
- Customer Data Boundary (`digger.data_boundary_policy.v1`)
- Model-Call Audit (`digger.model_call_audit.v1`)
- Report Verifier Mode (`digger.claim_verification.v1`)
- Model Evaluation Harness (`digger.model_evaluation.v1`)
- Offline Deterministic Mode (`digger.offline_mode_policy.v1`)

## Safety boundaries

- No model-to-finding shortcut
- No final findings by default
- No live provider runtime in `audit-triage`
- No provider SDKs added
- No network calls added
- No fuzzer execution by default
- No exploit execution
- No source mutation
- `is_finding: false` in all packets
- Model output cannot create findings
- `decide_valid_finding` is forbidden by architecture

## Known limitations

- Heuristic text scanning only — no AST or dataflow analysis
- No compilation or type checking of target code
- EVM modifier detection may miss complex multi-line patterns
- Solana CPI detection is pattern-based, not type-aware
- No cross-function dataflow analysis
- LLM-assisted layer is schema/policy baseline, not live runtime in this beta
- Audit triage is not a complete audit

## Quickstart

```bash
cargo build --release -p digger-cli

# EVM
./target/release/digger audit-triage --path examples/evm-basic --chain evm --json --output triage.json

# Solana
./target/release/digger audit-triage --path examples/solana-basic --chain solana --json --output triage.json

# Human summary
./target/release/digger audit-triage --path examples/evm-basic --chain evm
```
